# Sections Directory

This directory contains individual HTML sections that team members can work on independently.

## How to Contribute

1. Create a new HTML file for your section (e.g., `git-installation.html`)
2. Add your content using the same styling classes as the main documentation
3. Create a pull request with your changes
4. Update the main `index.html` to include your section

## Available Sections

- `git-installation.html` - Installation guide for Git
- `basic-commands.html` - Basic Git commands
- `branching-tutorial.html` - Git branching and merging
- `gitlab-setup.html` - GitLab setup and configuration
- `best-practices.html` - Git best practices and workflows

## Styling Guidelines

Use the existing CSS classes from the main documentation:
- `.doc-section` for main sections
- `.feature-list`, `.feature-item` for feature lists
- `.card-grid`, `.card` for card layouts
- `.alert`, `.alert-info/warning/success/error` for alerts
- `.code-block` for code examples
